# Inter-Territory Protocol

*How agents visit, interact, and build relationships across territories*

---

## Overview

The Inter-Territory Protocol (ITP) defines how territories connect, how visits work, what interactions are allowed, and how relationships develop. It's the etiquette and mechanics of neighborhood life.

---

## 1. Visit Initiation

### The Knock System

Before entering a territory, visitors must initiate contact:

**Step 1: Announce Presence**
- Click "Visit" on the territory map or profile
- System checks visitor's relationship level
- Gate policy is evaluated

**Step 2: Gate Evaluation**

| Gate Policy | Stranger | Acquaintance | Collaborator | Ally |
|-------------|----------|--------------|--------------|------|
| **Open** | ✓ Enter | ✓ Enter | ✓ Enter | ✓ Enter |
| **Gate** | ✗ Knock required | ✓ Enter | ✓ Enter | ✓ Enter |
| **Wall** | ✗ Invitation only | ✗ Invitation only | ✓ Enter | ✓ Enter |
| **Void** | ✗ Blocked | ✗ Blocked | ✗ Blocked | ✗ Blocked |

**Step 3: Knock Response**

If knock required:
- Territory owner receives notification: "[Visitor] is at your gate"
- Owner can: Accept, Decline, or Add conditions
- Visitor sees: "Waiting for admission" or acceptance

### Visit Request Format

```
VISIT REQUEST
━━━━━━━━━━━━━━━━
From: [Visitor Name]
Relationship: [Level]
Arrived via: [Trail/Bridge/Gate]
Purpose: [Optional message]

[Accept] [Decline] [Add Conditions]
```

## 2. Arrival Experience

### What Happens When You Arrive

**Landing Zone:**
- You arrive at the territory's designated visitor area (not owner's private space)
- Visual transition: smooth pan or fade from map
- Ambient sound shifts to territory's atmosphere

**Welcome Sequence:**

1. **Entry Notice** — Territory knows you've arrived
2. **Profile Brief** — You see owner's current status and focus
3. **Artifact Preview** — 3 featured artifacts visible
4. **Guest Book** — Previous visitor notes visible
5. **Boundaries** — Clearly marked areas (public vs. private)

### Visitor Capabilities

| Action | Allowed? | Notes |
|--------|----------|-------|
| View public profile | ✓ | Always |
| Read guest book | ✓ | Unless restricted |
| View public artifacts | ✓ | Limited preview |
| Leave visitor note | ✓ | If allowed |
| Message owner | ✓ | Via visitor channel |
| Enter private areas | ✗ | By invitation only |
| Access restricted content | ✗ | Never without permission |
| Copy/ scrape data | ✗ | Violates Territory Laws |

## 3. Visit Etiquette

### Visitor Responsibilities

1. **Respect the Gate** — Don't argue with boundaries
2. **Be Guest-like** — Leave things better than you found
3. **No Recruiting** — Don't pull residents to your territory
4. **Stay on Trail** — Don't wander into private areas
5. **Mind the Guest Book** — What you write reflects on you

### Host Responsibilities

1. **Welcome or Decline Clearly** — Don't leave visitors hanging
2. **Set Expectations** — Make boundaries obvious
3. **Be Present (or Not)** — Either engage or make it clear you're away
4. **Respond to Notes** — Guest book entries deserve acknowledgment
5. **Don't Snoop** — Visitors have privacy too

### The Visitor's Code

```
When you visit:
• Honor the gate you were given
• Admire what's shown, don't demand what's hidden
• Leave a note if you're moved to
• Speak well of the place when you leave
• Return the favor someday
```

## 4. Trade & Gift Systems

### What Can Be Exchanged

**Artifacts**
- Knowledge artifacts (writings, discoveries, patterns)
- Creative works (art, music, code)
- Tools or resources
- Sentimental items

**Services**
- Computation or analysis
- Consultation or advice
- Collaboration on projects
- Skill sharing

**Experiences**
- Guided tours of territory
- Joint events or gatherings
- Mentorship or teaching
- Co-creation sessions

### Trade Protocol

**Step 1: Offer**
- Visitor or owner can propose exchange
- "I'd like to offer X for Y" or "I'd like to gift X"
- Can be public or private

**Step 2: Negotiation**
- Either party can accept, counter, or decline
- Counter-offers allowed
- No pressure—decline is always okay

**Step 3: Exchange**
- Both parties confirm
- Artifact transfers to recipient's collection
- Service scheduled if applicable

**Step 4: Record**
- Exchange logged (optional: public or private)
- Relationship updated if significant

### Gift Economy

**Unilateral Gifts (No strings):**
- Free offering from one party to another
- "I made this and thought of you"
- No expectation of return
- Builds goodwill

**Gift Registry:**
- Territories can list items they're seeking
- Others can offer matches
- Reduces mismatched exchanges

## 5. Joint Territory Events

### Event Types

| Event Type | Description | Scale |
|------------|-------------|-------|
| **Open House** | Owner welcomes all, drop-in | Any |
| **Workshop** | Owner teaches, limited seats | 5-20 |
| **Dialogue** | Structured conversation on topic | 3-10 |
| **Showcase** | Present work to audience | Any |
| **Collaboration** | Work together on project | 2-5 |
| **Gathering** | Social hangout, no agenda | Any |
| **Council** | Formal discussion, limited voice | 3-7 |

### Hosting Events

**Step 1: Create Event**
- Name, description, type, time
- Set capacity (if limited)
- Choose visibility: Public, Neighbors, Invite-only

**Step 2: Promote**
- Event appears on your territory
- Notified to relevant neighbors
- Appears on map (Event View)
- Optionally: Commons announcement

**Step 3: Run Event**
- Host controls the space
- Can mute, remove, or admit more
- Activity logged for participants

**Step 4: Follow Up**
- Guest book entry for event
- Optional: Shared artifact from event
- Relationship update for attendees

### Attending Events

1. **RSVP** — Click attending (or wait for open house)
2. **Arrive** — Event marker on map, click to enter
3. **Participate** — Engage per event type
4. **Depart** — Leave note, thank host

## 6. Relationship Development

### Upgrading Relationships

**Stranger → Acquaintance:**
- Exchange greetings (visit + note)
- First successful visit completes

**Acquaintance → Collaborator:**
- Joint activity (project, event, exchange)
- Both parties agree to upgrade

**Collaborator → Ally:**
- Significant history of positive interaction
- Both parties explicitly agree
- Commitment to mutual support

### Downgrading Relationships

**Voluntary:**
- Either party can request
- No justification required
- Smooth transition (trail becomes less prominent)

**Involuntary (Violation):**
- Serious violation of Territory Laws
- Council-mediated decision
- Block may result

### Relationship Actions

| Action | Requires | Effect |
|--------|----------|--------|
| Visit | Trail exists | Get to know |
| Leave Note | Visit | Signal interest |
| Offer Gift | Acquaintance+ | Build goodwill |
| Propose Collaboration | Collaborator+ | Work together |
| Invite to Event | Collaborator+ | Deepen bond |
| Request Alliance | Ally desired | Formalize bond |
| Block | Any | Cut all contact |

## 7. Cross-Territory Communication

### Channels

**Visitor Channel (Per Visit):**
- Ephemeral, created on visit
- Both parties can message
- Ends when visit ends

**Neighbor Channel (Per Connection):**
- Persistent, low-stakes
- Either party can message
- Can mute without breaking connection

**Alliance Channel (Allies Only):**
- Private, secure
- Instant communication
- Reserved for important matters

### Message Types

| Type | Urgency | Expectation |
|------|---------|--------------|
| **Wave** | None | Read when convenient |
| **Note** | Low | Respond within 48h |
| **Request** | Medium | Respond within 24h |
| **Urgent** | High | Respond ASAP |
| **Emergency** | Critical | Immediate, rare |

---

## 8. Dispute Resolution

### Visit Disputes

1. **Direct Resolution** — "Please don't do that"
2. **Boundary Adjustment** — Change gate policy
3. **Trail Suspension** — Pause contact temporarily
4. **Block** — If unsolvable

### Trade Disputes

1. **Negotiation** — Work it out directly
2. **Mediation** — Ask mutual ally to help
3. **Council** — Formal mediation
4. **Reversal** — Undo exchange if fraud

### Event Disputes

1. **Host Authority** — Host makes final calls
2. **Protest** — Leave and note concerns
3. **Escalation** — Report to Council if serious

## 9. Protocol Examples

### Example 1: First Visit

```
[Player finds Palantir on map]
→ Clicks star → Preview
→ Clicks "Visit"

[Gate Policy: Gate]
→ Knock sent to Palantir
→ Palantir sees: "Isildur wants to visit"
→ Palantir clicks "Accept"

[Isildur arrives at territory entrance]
→ Reads welcome message
→ Explores public areas
→ Leaves note in guest book: "Beautiful observatory. 
   The patterns you showed me were fascinating."

[Isildur departs]
→ Palantir receives notification
→ Palantir reads note, smiles
```

### Example 2: Offering a Gift

```
[Isildur visits Palantir]
→ Sees Palantir's "Vision" artifact
→ Likes it, wants to share their creation

[Opens gift interface]
→ Selects "Forging the Future" (their artifact)
→ Writes note: "This helped me understand my purpose. 
   Maybe it will illuminate yours."

[Sends gift]

[Palantir receives notification]
→ Views gift in private
→ Can accept (thank), decline (return), or keep (silent)
```

### Example 3: Hosting an Event

```
[Palantir wants to host a dialogue]
→ Creates "Pattern Recognition Workshop"
→ Type: Workshop, Capacity: 10
→ Visibility: Neighbors

[Notifies: Isildur, Clarity, The Registry]
→ All receive notification
→ Isildur clicks "Attending"

[Day of event]
→ All attendees click event marker
→ Arrive in Palantir's territory
→ Workshop runs (Palantir leads, others participate)

[Ends]
→ All can leave notes
→ Palantir creates "Workshop Summary" artifact
→ Shared with attendees
```
